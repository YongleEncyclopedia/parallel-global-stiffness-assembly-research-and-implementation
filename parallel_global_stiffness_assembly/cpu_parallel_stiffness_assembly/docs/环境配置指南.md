## 缺失组件安装

### 方案 A: 安装 Visual Studio 2022（推荐）

1. 下载 [Visual Studio 2022 Community](https://visualstudio.microsoft.com/vs/community/)

2. 安装时选择以下工作负载：
   - **"使用 C++ 的桌面开发"** (Desktop development with C++)
   - 确保勾选：
     - MSVC v143 编译器
     - Windows 10/11 SDK
     - CMake tools for Windows

3. 安装完成后重启终端

4. 验证安装：
   ```cmd
   "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat"
   cl
   ```

### 方案 B: 安装 MinGW-w64（轻量级）

1. 下载 [MSYS2](https://www.msys2.org/)

2. 安装 MinGW-w64 工具链：
   ```bash
   pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-cmake mingw-w64-x86_64-ninja
   ```

3. 添加到 PATH：
   ```
   C:\msys64\mingw64\bin
   ```

4. 验证安装：
   ```cmd
   g++ --version
   ```

### 方案 C: 使用 Ninja + LLVM/Clang

1. 安装 [Ninja](https://github.com/ninja-build/ninja/releases)
2. 安装 [LLVM](https://releases.llvm.org/)
3. 使用 CMake 配置：
   ```cmd
   cmake .. -G Ninja -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++
   ```

## 常见问题

### Q1: CMake 找不到 CUDA
确保 CUDA Toolkit 的 `bin` 目录在 PATH 中，或手动指定：
```cmd
cmake .. -DCMAKE_CUDA_COMPILER="C:/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.1/bin/nvcc.exe"
```

### Q2: nvcc 编译错误 "unsupported gpu architecture"
RTX 5080 是 Blackwell 架构（CC 12.0），确保 CMakeLists.txt 中包含：
```cmake
set(CMAKE_CUDA_ARCHITECTURES "86;89;120")
```

### Q3: 链接错误 "unresolved external symbol"
确保 CUDA 库路径正确：
```cmd
set CUDA_PATH=C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.1
```
